# Процесс выполнения
Выполняй задачи по `docs/tasks.md` строго по порядку: один подпункт — один шаг.
После каждого шага:
- обновляй `docs/tasks.md` (галочки и/или уточнения),
- кратко перечисляй, что изменилось и где,
- если всплыла ошибка — сначала локализуй причину и только потом правь.

# Nuxt 4: структура и роутинг
Проект на Nuxt 4, соблюдай актуальные подходы:
- корневой компонент: `app/app.vue` и он должен рендерить `<NuxtPage />`,
- страницы лежат в `pages/` (например, `/` = `pages/index.vue`),
- для UI секций используй компоненты из `components/`,
- не смешивай “контент страницы” в `app/app.vue`, он только оболочка приложения,
- любые проблемы с 404 сначала проверяй наличие `pages/index.vue` и то, что dev запущен из корня проекта.

# Стиль кода
Выноси компоненты в `components/`, не раздувай страницы.
Соблюдай SOLID и KISS.
Не пиши комментарии в коде совсем.

# Tailwind v4 и темы
Используй Tailwind v4.
Не переписывай `nuxt.config.ts` ради стилизации.
Цвета храни как CSS-переменные:
- объявляй в `:root` (и альтернативные значения для тёмной темы),
- затем маппь в `@theme`,
- для классов используй `bg-[color:var(--...)]`, `text-[color:var(--...)]`, т.д.
Сделай светлую/тёмную тему и переключение темы.

# Контент и блог

`/content/` не добавлять в `.gitignore`.
